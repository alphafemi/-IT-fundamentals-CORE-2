Comprehensive walkthrough of the **Microsoft Management Console (MMC)** and its most important snap-ins. For the CompTIA Core 2 exam, knowing what each snap-in does, how to launch it individually, and when to use it is essential.



---

## 1. Microsoft Management Console (MMC) Overview

| Command | Purpose |
|---------|---------|
| `mmc.exe` | Launches an empty Microsoft Management Console |
| **Snap-in** | A utility added to MMC (e.g., Event Viewer, Disk Management) |
| **.msc file** | A saved console configuration (e.g., `eventvwr.msc`) |

**Process:**
1. Run `mmc` (empty console)
2. File → Add/Remove Snap-in
3. Choose snap-ins → Add → Select local or remote computer
4. File → Save As (creates a custom `.msc` file)

> **Exam Tip:** MMC allows you to build a **customized, single-view** of multiple administrative tools.

---

## 2. Key MMC Snap-ins (and their individual commands)

| Snap-in | Individual Command | Purpose |
|---------|-------------------|---------|
| **Event Viewer** | `eventvwr.msc` | View Windows logs (Application, Security, Setup, System) |
| **Disk Management** | `diskmgmt.msc` | Manage partitions, drive letters, volumes |
| **Task Scheduler** | `taskschd.msc` | Schedule automated tasks (time/date/recurring) |
| **Device Manager** | `devmgmt.msc` | View/manage hardware drivers |
| **Certificate Manager** | `certmgr.msc` | View/manage digital certificates |
| **Local Users and Groups** | `lusrmgr.msc` | Manage local user accounts and groups |
| **Performance Monitor** | `perfmon.msc` | Track resource metrics over time (CPU, memory, disk) |
| **Local Group Policy Editor** | `gpedit.msc` | Edit policies for a single computer |
| **Group Policy Management** | `gpmc.msc` | Manage policies across Active Directory domain |

> **Note:** `gpmc.msc` requires Active Directory (enterprise). `gpedit.msc` works on standalone Pro/Enterprise systems.

## 3. Detailed Breakdown: Each Snap-in

### Event Viewer (`eventvwr.msc`)
**Log categories:**
- Application
- Security
- Setup
- System

**Event levels:**
| Level | Description |
|-------|-------------|
| Information | Normal operation |
| Warning | Potential issue |
| Error | Problem occurred |
| Critical | System failure |
| Success Audit | Security access granted |
| Failure Audit | Security access denied |

**Key feature:** Filter logs by level, source, user, computer, or time.

### Disk Management (`diskmgmt.msc`)

**What you can see:**
- Partitions and drive letters
- File systems (NTFS, FAT32, exFAT)
- Health status (Healthy, Failed, etc.)
- Partition types (Boot, Page file, Crash dump, Data)

**What you can do (right-click):**
| Action | Description |
|--------|-------------|
| Change drive letter and paths | Assign/modify drive letters |
| Shrink volume | Reduce partition size |
| Extend volume | Expand partition (requires free space) |
| Mirror volume | RAID 1 across physical drives |

> **Warning:** Changes here can cause data loss. Always have a backup.

### Task Scheduler (`taskschd.msc`)

**Components of a task:**
| Tab | Configures |
|-----|-------------|
| Triggers | When task runs (time, date, recurring) |
| Actions | What happens (run program, send email) |
| Conditions | Requirements (AC power, network connection) |
| Settings | Additional options (retry on failure, stop if running too long) |

**Default view:** Shows tasks that ran in last 24 hours (can change to 7 or 30 days).

### Device Manager (`devmgmt.msc`)

**Organization:** Hardware grouped by category (Display adapters, Keyboards, Monitors, Printers, etc.)

**Right-click options:**
| Option | Purpose |
|--------|---------|
| Update driver | Install newer driver version |
| Disable device | Turn off hardware without removing driver |
| Uninstall device | Remove driver completely |
| Properties | View driver details, events, resources |

**Driver compatibility:** Drivers are generally OS-specific (Windows 10 driver for Windows 10).

### Certificate Manager (`certmgr.msc`)

**Purpose:** View and manage digital certificates

**Common use:** View **Trusted Root Certification Authorities** (CAs) that your browser trusts.

### Local Users and Groups (`lusrmgr.msc`)

**Built-in users:**
- Administrator
- Guest
- DefaultAccount

**Built-in groups (examples):**
| Group | Privileges |
|-------|------------|
| Administrators | Full system control |
| Users | Standard rights (cannot install software/system changes) |
| Backup Operators | Can back up/restore files |
| Power Users | Legacy group (limited admin rights) |

**Action:** Right-click → New User to add accounts.

### Performance Monitor (`perfmon.msc`)

**Capabilities:**
- Collect **hundreds** of metrics over hours/days/weeks
- Set **alerts** when resource thresholds exceeded
- Create **reports** with graphical views

**Common counters:**
| Category | Examples |
|----------|----------|
| Processor | % Processor Time, Interrupts/sec |
| Memory | Available MBytes, Pages/sec |
| Physical Disk | % Disk Time, Avg. Disk Queue Length |
| Network | Bytes Total/sec, Packets/sec |

**How to add counters:**
1. Click the **+** (plus sign)
2. Select category (e.g., Processor)
3. Choose specific counters
4. Click Add → OK

> **Use case:** Troubleshoot intermittent problems that occur when you're not watching.

### Group Policy Editor

| Tool | Command | Environment |
|------|---------|-------------|
| Local Group Policy Editor | `gpedit.msc` | Single computer (Pro/Enterprise) |
| Group Policy Management Console | `gpmc.msc` | Active Directory domain (enterprise) |

**Two main categories:**
1. **Computer Configuration** – Applies to the machine regardless of user
2. **User Configuration** – Applies to user accounts

**Example policy:** Hide and disable all items on the desktop (User Configuration → Administrative Templates → Desktop)

## 4. Quick Reference: Which Tool for Which Task?

| Task | Best Tool |
|------|-----------|
| See why an application crashed yesterday | Event Viewer |
| Free up space by shrinking a partition | Disk Management |
| Automatically run a backup every night at 2 AM | Task Scheduler |
| Find out if a network driver is causing problems | Device Manager |
| See which certificates your browser trusts | Certificate Manager |
| Add a new local user account | Local Users and Groups |
| Track CPU usage over a week to find intermittent spikes | Performance Monitor |
| Prevent users from changing desktop background | Group Policy Editor (`gpedit.msc`) |
| Create a custom console with Event Viewer + Disk Management | MMC (`mmc.exe`) |

## 5. Sample CompTIA-Style Questions

> *A technician needs to view the System log to find an error that occurred during last night's backup. Which command should they run?*
>
> A. `diskmgmt.msc`  
> B. `eventvwr.msc`  
> C. `perfmon.msc`  
> D. `taskschd.msc`  

**Answer:** B (Event Viewer)

> *A user reports their computer becomes unresponsive every day at 3 PM. Which tool should you use to investigate scheduled tasks that might be causing the issue?*
>
> A. `certmgr.msc`  
> B. `gpedit.msc`  
> C. `taskschd.msc`  
> D. `devmgmt.msc`  

**Answer:** C (Task Scheduler)

> *Which MMC snap-in would you use to assign the letter X: to an external USB drive?*
>
> A. Device Manager  
> B. Disk Management  
> C. Performance Monitor  
> D. Local Users and Groups  

**Answer:** B (Disk Management)

> *A technician wants to create a custom console that contains only Device Manager and Event Viewer. What should they do first?*
>
> A. Run `devmgmt.msc`  
> B. Run `eventvwr.msc`  
> C. Run `mmc.exe`  
> D. Run `gpmc.msc`  

**Answer:** C (Launch an empty MMC, then add snap-ins)
