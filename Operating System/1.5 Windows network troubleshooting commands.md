This is an outstanding, comprehensive guide to **Windows network troubleshooting commands** – essential knowledge for the CompTIA Core 2 exam and real-world IT support. 
Here is a structured study guide based on your text.

---

## 1. IP Configuration (`ipconfig`)

| Command | Information Provided |
|----------|---------------------|
| `ipconfig` | IPv4/IPv6 address, subnet mask, default gateway |
| `ipconfig /all` | Everything above + hostname, DNS servers, DHCP status, MAC address |

**Use case:** First command to run when troubleshooting network issues. Shows how the system is configured.

---

## 2. Connectivity Testing (`ping`)

**Protocol used:** ICMP (Internet Control Message Protocol)

| Feature | Description |
|---------|-------------|
| Default count | 4 packets sent |
| What it shows | Reply status, bytes, response time (ms), TTL |
| Summary | Packets sent/received, % loss, min/max/avg round-trip time |

**Example output interpretation:**
- 0% loss = good connectivity
- High response times = latency issues
- Request timeout = no connectivity

---

## 3. Network Statistics (`netstat`)

| Command | What It Shows |
|---------|----------------|
| `netstat` | Active TCP/UDP connections, local/foreign addresses, state |
| `netstat -a` | All active connections (includes listening ports) |
| `netstat -b` | Shows executable/binary for each connection (**requires admin**) |
| `netstat -n` | Shows IP addresses (no DNS resolution) |

**Connection states you may see:**
- ESTABLISHED
- CLOSE-WAIT
- LISTENING
- TIME-WAIT

> **Exam Tip:** `netstat -b` is very useful to identify *which application* is making a network connection.

---

## 4. DNS Lookup (`nslookup`)

| Command | Purpose |
|---------|---------|
| `nslookup www.professormesser.com` | Resolve FQDN to IP address(es) |
| `nslookup` (enter, then interactive) | Interactive mode for advanced queries |

**What it shows:**
- Default DNS server being used
- IP address(es) associated with the domain name

> **Note:** Multiple IP addresses for one FQDN indicates redundancy (not an error).

---

## 5. Windows NET Commands

| Command | Purpose |
|---------|---------|
| `net view \\SERVERNAME` | View available shares on a server |
| `net use H: \\SERVERNAME\SHARENAME` | Map a network drive to letter H: |
| `net user username /domain` | View user account details (password expiry, groups, login script) |

**Net user information shown:**
- Full name, account active status
- Password last set / expires
- Login script, home directory
- Group memberships

---

## 6. Route Tracing (`tracert`)

**How traceroute works:**

| TTL Value | What Happens |
|-----------|--------------|
| TTL = 1 | First router decrements to 0 → sends "Time to Live Exceeded" back |
| TTL = 2 | Reaches second router (first decrements 2→1, second 1→0) |
| TTL = 3, 4, 5... | Continues until destination reached |

**Key concept:** In IPv4, **TTL (Time to Live) = number of hops** (not seconds or time of day).

**Command:** `tracert 8.8.8.8`

**What it shows:**
- Each hop (router) along the path
- Response time for each hop (3 attempts per hop)
- IP address and (if available) hostname of each router

> **Limitation:** Firewalls may filter ICMP, causing incomplete results.

---

## 7. Path Analysis (`pathping`)

**What it does:** Combines `tracert` + `ping` in one utility.

| Phase | Action |
|-------|--------|
| **Phase 1** | Runs traceroute to map the path |
| **Phase 2** | Sends round-trip tests to each hop (typically 300+ seconds) |

**Output includes:**
- Hop-by-hop latency statistics
- Packet loss percentage per hop
- Address information for each router

**Example use case:** Identifying *which hop* is causing packet loss or high latency.

## 8. Quick Reference: Network Commands Summary

| Command | Primary Purpose | Example |
|---------|----------------|---------|
| `ipconfig` | View IP configuration | `ipconfig /all` |
| `ping` | Test reachability | `ping 192.168.1.1` |
| `netstat` | View connections/ports | `netstat -b` (admin) |
| `nslookup` | DNS resolution | `nslookup google.com` |
| `net view` | List network shares | `net view \\server` |
| `net use` | Map network drive | `net use H: \\server\share` |
| `net user` | User account info | `net user jsmith /domain` |
| `tracert` | Trace route (hops) | `tracert 8.8.8.8` |
| `pathping` | Trace + test each hop | `pathping 8.8.8.8` |

---

## 9. Comparison: `ping` vs `tracert` vs `pathping`

| Feature | `ping` | `tracert` | `pathping` |
|---------|--------|-----------|------------|
| Tests connectivity | Yes | Yes | Yes |
| Shows hops | No | Yes | Yes |
| Tests each hop | No | Basic (time only) | Yes (loss + time) |
| Time to complete | Seconds | Seconds | Minutes (300+ sec) |
| Best for | Quick up/down check | Path discovery | Detailed hop analysis |

## 10. Sample CompTIA-Style Questions

> *A user reports intermittent network slowness. You need to identify which router along the path is causing packet loss. Which command should you use?*
>
> A. `ping`  
> B. `tracert`  
> C. `pathping`  
> D. `netstat -b`  

**Answer:** C (`pathping` tests each hop for loss over time)

> *A technician wants to see which application is establishing outbound connections on a Windows computer. Which command requires administrator privileges?*
>
> A. `ipconfig /all`  
> B. `netstat -b`  
> C. `nslookup`  
> D. `ping`  

**Answer:** B (`netstat -b` shows executables and requires elevated prompt)

> *A user cannot access a network share. Which command would map drive H: to \\fileserver\documents?*
>
> A. `net view H: \\fileserver\documents`  
> B. `net use H: \\fileserver\documents`  
> C. `net user H: \\fileserver\documents`  
> D. `net share H: \\fileserver\documents`  

**Answer:** B (`net use` maps network drives)

> *A technician runs `ping 8.8.8.8` and gets "Request timed out" for all four packets. What is the most likely cause?*
>
> A. The DNS server is down  
> B. No connectivity to 8.8.8.8  
> C. The local IP address is incorrect  
> D. DHCP is not enabled  

**Answer:** B (No response indicates lack of connectivity to that destination)

> *Which protocol does `ping` and `tracert` primarily use?*
>
> A. TCP  
> B. UDP  
> C. ICMP  
> D. ARP  

**Answer:** C (ICMP - Internet Control Message Protocol)

## 11. Troubleshooting Scenarios (Exam Prep)

| Problem | Command(s) to Run First |
|---------|------------------------|
| "I have no internet access" | `ipconfig` (check IP, gateway), `ping 8.8.8.8` |
| "The network is slow" | `pathping` (find loss/latency per hop) |
| "I can't reach a specific server" | `ping server`, `tracert server` |
| "I need to see what's listening on port 80" | `netstat -a` |
| "What DNS server am I using?" | `ipconfig /all` or `nslookup` |
| "What shares are on that server?" | `net view \\servername` |
| "When does this user's password expire?" | `net user username /domain` |
| "Which app is using all my bandwidth?" | `netstat -b` (run as admin) |
