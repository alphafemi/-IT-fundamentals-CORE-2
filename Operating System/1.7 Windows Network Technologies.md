## 1. Windows Shares: Sharing Resources Across the Network

| Concept | Description |
|---------|-------------|
| **Windows share** | A file, folder, printer, or resource made available across the network |
| **Map a drive** | Assign a drive letter to a network share (e.g., X: → \\server\share) |
| **net use** | Command-line method to map a drive |
| **Hidden share (Admin share)** | Share name ending with `$` (e.g., `ADMIN$`, `C$`) – does not appear in browse lists |

### Hidden/Administrative Shares

| Share Name | Purpose |
|------------|---------|
| `ADMIN$` | Remote administration (points to Windows folder) |
| `C$`, `D$`, etc. | Root of each drive |
| `IPC$` | Inter-process communication (remote administration) |

> **Key point:** Hidden shares do NOT appear in File Explorer or browse lists, but users who know the name can still access them (if they have permissions).

### Viewing All Shares on a Local Machine

**Path:** Computer Management → Shared Folders → Shares

This shows every share (including hidden/admin shares) on the device.

### Mapping a Network Drive

**GUI Method (File Explorer):**
1. Three dots (or right-click This PC) → Map network drive
2. Select drive letter
3. Browse for share or enter path (`\\server\share`)
4. Options: Reconnect at sign-in, Connect using different credentials

**Command-line Method:**
```cmd
net use X: \\server\share
net use X: \\server\share /persistent:yes   (reconnect at login)
```

**To disconnect:**
- GUI: Three dots → Disconnect network drive
- Command: `net use X: /delete`

## 2. Workgroups vs. Domains

| Feature | **Workgroup** | **Domain** |
|---------|---------------|------------|
| **Management** | Decentralized (each device manages itself) | Centralized (Active Directory) |
| **Authentication** | Local accounts on each device | Single sign-on with domain credentials |
| **Credential management** | Different username/password per device | One set of credentials for all resources |
| **Scalability** | Small (10-20 devices max) | Large (hundreds/thousands of devices) |
| **Typical environment** | Home, small office | Business, enterprise |
| **Central administration** | No | Yes (IT team manages all from one place) |
| **Server required** | No | Yes (Active Directory Domain Services) |

### Workgroup Characteristics
- Every device is **standalone**
- Each device maintains its **own** usernames and passwords
- Same user may need different credentials for different devices
- No central database

### Domain Characteristics
- **One set of credentials** for all resources
- Central management of users, devices, and policies
- Requires Active Directory Domain Services (AD DS) – a distributed database
- Enterprise environment standard

## 3. Checking Workgroup/Domain Status

**Location:** System applet (Settings → System → About, or Control Panel → System)

**What it shows:**
- Device name
- Domain or workgroup membership

**To change:**
1. Click "Domain or workgroup" (or "Rename this PC")
2. Click Change
3. Rename device, change to domain, or change to workgroup
4. Provide domain credentials when joining a domain

**Example dialog information:**
- Computer name
- Full computer name
- Domain (e.g., `company.local`)
- Workgroup (e.g., `WORKGROUP`)

## 4. Joining a Domain

### Requirements

| Requirement | Details |
|-------------|---------|
| **Windows edition** | **Pro, Enterprise, or Education** (Home edition cannot join a domain) |
| **Network connectivity** | Must reach domain controller |
| **Credentials** | Domain account with permission to join devices |
| **Correct time** | Time synchronization required for Kerberos authentication |

### Process
1. Go to System → About → Domain or workgroup
2. Click Change
3. Select "Domain" and enter domain name
4. Enter domain credentials when prompted
5. Restart computer

> **Exam Tip:** Windows **Home** editions cannot join a domain. This is a key distinction.

## 5. Printer Sharing

### Steps to Share a Printer
1. Settings → Bluetooth & Devices → Printers & scanners
2. Select the printer → Printer Properties
3. Click **Sharing** tab
4. Check "Share this printer"
5. Enter a share name
6. Click OK

### What happens
- Printer becomes available on the network
- Other users can connect to `\\computername\printername`
- Permissions can be set on the Security tab

## 6. Active Directory Domain Services (AD DS)

| Feature | Description |
|---------|-------------|
| **Database type** | Distributed (multiple copies across network) |
| **Stores** | Users, devices, printers, servers, permissions, security policies |
| **Authentication** | Kerberos (requires time synchronization) |
| **Management** | Group Policy, central user/device management |
| **Typical use** | Business/enterprise (not home) |

**Why not at home?** Requires separate server infrastructure; overkill for small networks.

## 7. Quick Reference: Share Types

| Share Type | Example | Visible in Browse? | Purpose |
|------------|---------|-------------------|---------|
| **Standard share** | `Documents` | Yes | User-created file/folder share |
| **Hidden share** | `SecretData$` | No | Administrative or hidden access |
| **Admin share** | `C$`, `ADMIN$` | No | Built-in remote management |
| **Printer share** | `HP_LaserJet` | Yes (if shared) | Network printer access |

## 8. Common Commands Summary

| Command | Purpose |
|---------|---------|
| `net use X: \\server\share` | Map drive letter to share |
| `net use X: /delete` | Disconnect mapped drive |
| `net share` | List local shares (command line) |
| `net view \\server` | View shares on a remote server |
| `hostname` | Display current computer name |

## 9. Sample CompTIA-Style Questions

> *A user wants to share a folder on their Windows 11 Home computer so that other users on the home network can access it. They want the share to appear in the network browse list. What should they do?*
>
> A. Create a share named `Folder$`  
> B. Create a standard share without a dollar sign  
> C. Join the computer to a domain first  
> D. Use `net use` to create the share  

**Answer:** B (Standard share without `$` appears in browse list)

> *An IT administrator needs to remotely manage a Windows 11 Pro computer. Which administrative share would allow access to the Windows folder?*
>
> A. `C$`  
> B. `ADMIN$`  
> C. `IPC$`  
> D. `Windows$`  

**Answer:** B (`ADMIN$` points to the Windows folder for remote administration)

> *A small business has 5 computers in a workgroup. The owner wants to centralize user management so employees only need one password. What should they implement?*
>
> A. Continue using workgroup mode  
> B. Upgrade all computers to Windows 11 Home  
> C. Implement a Windows domain with Active Directory  
> D. Use hidden shares for all resources  

**Answer:** C (Domain with Active Directory provides centralized authentication)

> *A technician is trying to join a Windows 11 computer to a domain but cannot find the option. What is the most likely reason?*
>
> A. The computer is running Windows 11 Home  
> B. The computer is not connected to the internet  
> C. The computer name is already in use  
> D. The computer has a hidden share configured  

**Answer:** A (Windows Home editions cannot join a domain – requires Pro/Enterprise)

> *Which command would map drive S: to a share named `\\fileserver\sales`?*
>
> A. `net share S: \\fileserver\sales`  
> B. `net use S: \\fileserver\sales`  
> C. `net view S: \\fileserver\sales`  
> D. `net config S: \\fileserver\sales`  

**Answer:** B (`net use` maps network drives)

## 10. Exam Tips Summary

| Scenario | Solution/Knowledge |
|----------|-------------------|
| Share doesn't appear in browse list | Check for `$` at end of share name (hidden share) |
| Need to see all shares (including hidden) | Computer Management → Shared Folders → Shares |
| Computer can't join domain | Check if running Windows Home edition |
| Multiple passwords for different devices | Likely a workgroup – consider domain |
| One password for all resources | Domain with Active Directory |
| Share a printer | Printer Properties → Sharing tab |
| Map a drive from command line | `net use X: \\server\share` |
| Disconnect a mapped drive | `net use X: /delete` |
