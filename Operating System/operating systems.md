**Core Functions of an OS)**
- **Hardware Management:** Controls interaction between CPU, memory, storage, and I/O devices (keyboard, monitor, printer).
- **Application Platform:** Executables are written for a specific OS.
- **User Interface:** Allows humans to input information, process it, and receive output.
- **File Management:** Store, remove, and modify files on storage devices.
- **Memory Management:** Handles RAM and swap files (moving data between memory and storage).
- **Utilities:** Built-in tools to keep the system running efficiently.

**Major Operating Systems Compared**

| OS | Key Characteristic | Pros | Cons |
|---|---|---|---|
| **Windows** | Most popular, broad industry support | Large hardware/software support, many developers, vast community help | Primary target for attackers, driver quality varies by manufacturer |
| **Linux** | Free, open-source, many distributions | No cost, works on almost any hardware, large community support | New hardware compatibility issues, no single company for tech support |
| **macOS** | Runs only on Apple hardware, ease of use | Excellent UI, high hardware-software compatibility, designed with security in mind | Must buy Apple hardware (higher cost), fewer users than Windows |
| **Chrome OS** | Browser-centric (Chrome), lightweight | Simple, low overhead, runs on many manufacturers' laptops | Requires constant network connectivity for most apps |
| **iOS/iPadOS** | Apple mobile OS, closed source | Tight hardware/software integration, curated App Store | Only runs on Apple mobile devices, dev requires macOS & Apple approval |
| **Android** | Open-source (Linux-based), multiple manufacturers | Runs on many devices, dev on Windows/macOS/Linux, multiple app stores | Fragmentation across devices/manufacturers |

**Critical Common Considerations**
- **End of Life (EOL):** Different vendors have different support lifespans (Apple vs. Microsoft).
- **Updates:** Essential for both efficiency and security patches.
- **Cross-Platform Data:** Documents, spreadsheets, media can move between OSes.
- **Executables are NOT cross-platform:** A Windows `.exe` won't run on Linux or macOS unless using emulation/compatibility layers (which your text correctly notes is not native).
- **Web-based apps:** Increasingly bypass OS limitations by running in any browser.

**One minor clarification** (based on standard industry knowledge):  
While you correctly note that Chrome OS is based on the Linux kernel, modern Chrome OS also includes the ability to run Linux applications in a container (Crostini) and Android apps via the Play Store, reducing its reliance on pure cloud connectivity compared to its earliest versions.
