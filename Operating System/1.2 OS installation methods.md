## 1. Boot Methods to Start an OS Installation

| Method | Description | Best For |
|--------|-------------|----------|
| **Bootable USB drive** | Specialized software makes USB bootable; motherboard must support USB boot | Single machines, personal use |
| **Network boot (PXE)** | Preboot Execution Environment; BIOS looks for a PXE server on local network | Large organizations, no physical media needed |
| **Internet boot** | Minimal local code downloads rest from internet (Linux net install, macOS Recovery, Windows Update) | Remote installations, recovery scenarios |
| **Optical disk (CD/DVD)** | Older method; ISO image format standardizes optical disk contents | Legacy systems |
| **ISO image** | Disk image file containing entire OS; can be mounted as virtual drive or written to USB | Virtualization, external drives |
| **Internal partition** | Installation files copied to a local partition; boot from that partition | Advanced multi-boot setups |

## 2. Installation Types

| Installation Type | What It Does | Data/Apps Preserved? |
|------------------|--------------|----------------------|
| **Clean install** | Wipes partition completely; reinstalls OS from scratch | No |
| **In-place upgrade** | Updates OS while keeping applications and data | Yes |
| **Image deployment** | Pre-configured OS + apps + settings copied to multiple machines | Varies (image replaces everything) |
| **Repair installation** | Overwrites OS files only; leaves user files and documents | Yes (user data preserved) |
| **Recovery partition** | Hidden partition with OS installation files; accessible without external media | Varies |

## 3. Partitioning Fundamentals

### What is a Partition?
A logical section of a storage drive designated to store data.

### Partition Table Styles

| Feature | **GPT** (GUID Partition Table) | **MBR** (Master Boot Record) |
|---------|-------------------------------|------------------------------|
| **Requires** | UEFI BIOS | Legacy BIOS |
| **Max partitions** | 128 | 4 primary (or 3 primary + 1 extended) |
| **Max drive size** | 9+ billion TB (Windows limits to 256 TB) | 2 TB |
| **Bootable partitions** | All partitions can be bootable | Only primary partitions are bootable |
| **Extended partitions** | Not needed | Can create 1 extended partition containing multiple logical drives (not bootable) |
| **Modern OS support** | Yes (Windows 10/11, modern Linux/macOS) | Legacy only |

### Key MBR Limitations (from your text)
- Only **4 primary partitions** per drive
- Only **one active bootable partition** at a time
- **Logical partitions** (inside extended partition) are **not bootable**

## 4. Formatting Options

| Format Type | What It Does | Speed | Security | Default in Windows 10/11? |
|-------------|--------------|-------|----------|---------------------------|
| **Quick format** | Creates new file table; does not check drive for errors; old data potentially recoverable | Fast | Low (data can be recovered) | Yes |
| **Full format** | Writes zeros across entire disk; erases all previous data; checks for bad sectors | Slow | High (data cannot be easily recovered) | No (requires diskpart) |

> **Security note:** Quick format only erases the *file table*, not the actual data. Recovery software can retrieve old files.

## 5. Advanced Deployment Concepts

| Concept | Description |
|---------|-------------|
| **Zero-touch deployment** | Fully automated installation with no user prompts; configures domain, email, and organization-specific settings automatically |
| **Remote network installation** | Installation files stored on network drive; no local media required |
| **Multi-boot system** | Multiple OSes (e.g., Windows + Linux) on same device; choose at boot |
| **Third-party drivers during install** | Required when installation program doesn't recognize storage or network hardware; loaded via prompt during setup |

---

## 6. Important Cautions

> **"This process of creating a partition or removing a partition can very easily delete your data."**

- Always verify which partition you are modifying
- Changing partitions can cause irreversible data loss
- Recovery partition is often **hidden** – users may not see it but it contains OS installation files

---

## Quick Reference: Which Method Should You Use?

| Scenario | Recommended Approach |
|----------|----------------------|
| Home user installing Windows on one PC | Bootable USB drive, clean install, quick format |
| IT department deploying 100 identical computers | Image deployment + PXE boot + zero-touch |
| Upgrading from Windows 10 to 11 keeping files | In-place upgrade |
| Computer won't boot and no installation media | Recovery partition |
| Secure disposal of a drive before selling | Full format (writes zeros) |
| Multi-boot Windows + Linux | Separate partitions (GPT recommended) |

## One Clarification

You wrote: *"If you enable the BIOS compatibility mode, it disables the ability to SecureBoot, which means that many of your newer operating systems will not work in a BIOS compatibility mode."*

This is correct. **SecureBoot** (a UEFI feature) verifies that bootloaders are signed by trusted authorities. Windows 11 **requires** SecureBoot and UEFI (not legacy BIOS compatibility mode). So systems with compatibility mode enabled cannot run Windows 11 natively.
