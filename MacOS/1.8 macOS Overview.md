This is an excellent, comprehensive guide to **macOS fundamentals** – essential knowledge for the CompTIA Core 2 exam, which expects you to be familiar with multiple operating systems, not just Windows.



---

## 1. macOS File Types and Extensions

| Extension | Full Name | Purpose | Windows Equivalent |
|-----------|-----------|---------|-------------------|
| `.dmg` | Apple Disk Image | Mountable drive image (similar to ISO) | `.iso` |
| `.pkg` | Installer Package | Software distribution installer | `setup.exe` or `.msi` |
| `.app` | Application Bundle | Executable application (contains multiple files) | `.exe` |

### Key Differences from Windows

| Windows | macOS |
|---------|-------|
| `.exe` (single executable file) | `.app` (bundle containing multiple files) |
| Program Files folder | Applications folder |
| `setup.exe` to install | `.pkg` installer or drag `.app` to Applications |
| Uninstall via Programs and Features | Delete `.app` file (drag to Trash) |

> **Exam Tip:** An `.app` file is a **bundle** – right-click → View Package Contents to see internal files. Users only see the single `.app` icon.

## 2. macOS Folder Structure

| Folder | Path | Purpose | Windows Equivalent |
|--------|------|---------|-------------------|
| **Applications** | `/Applications` | All installed applications | `C:\Program Files` |
| **Users** | `/Users/` | User home directories | `C:\Users` |
| **User Home** | `/Users/username` (or `~`) | Individual user's documents, settings | `C:\Users\username` |
| **Library (System)** | `/Library` | Shared support files (fonts, scripts, etc.) | `C:\Windows\System32` or ProgramData |
| **Library (User)** | `~/Library` | User-specific support files | `C:\Users\username\AppData` |
| **System** | `/System` | Core operating system files | `C:\Windows` |

> **Note:** The user's `Library` folder (`~/Library`) is **hidden by default** in Finder, but can be accessed directly.

**Path notation:**
- `/Users/professor` – absolute path
- `~/Library` – tilde (`~`) means user's home directory

## 3. macOS App Management

### App Store

| Feature | Description |
|---------|-------------|
| **Purpose** | Centralized repository for applications and updates |
| **Automatic updates** | Can be enabled to update apps automatically |
| **Manual updates** | App Store → Updates tab → Install individually |
| **Account view** | Click User ID (bottom left) to see all installed apps |

### Installing Applications

| Method | Description |
|--------|-------------|
| **App Store** | Download and install directly |
| **.pkg installer** | Runs installation script (similar to Windows setup) |
| **Drag to Applications** | Some `.app` files can be dragged directly to Applications folder |

### Uninstalling Applications

| Method | Description |
|--------|-------------|
| **Delete .app** | Drag `.app` from Applications to Trash (most common) |
| **Run uninstaller** | Some apps include `uninstaller.app` (e.g., Logitech software) |

> **Why this works:** The `.app` bundle contains all application files – deleting the bundle removes everything.

## 4. Apple ID vs. Managed Apple ID

| Feature | Apple ID | Managed Apple ID |
|---------|----------|------------------|
| **Purpose** | Personal use | Business/enterprise use |
| **Management** | User self-manages | Organization manages via Apple Business Manager |
| **Integration** | iCloud, App Store purchases | Active Directory, MDM (Mobile Device Manager) |
| **Use case** | Personal Apple devices | Corporate-owned devices |

### Apple Business Manager Features
- Integrates with Active Directory (same credentials as Windows)
- MDM integration for centralized device management
- Assign permissions, move apps, coordinate data access

## 5. Time Machine (Backup)

| Feature | Description |
|---------|-------------|
| **Frequency** | Automatic hourly backups |
| **Daily backups** | Past month |
| **Weekly backups** | All previous months |
| **Retention** | Oldest backups deleted when space is full |
| **Destination** | Time Machine server or external drive |

**Use case:** "Go back in time" to find a specific file version.

## 6. macOS Security

### Built-in Security
- Anti-malware features (built into OS, but no single management app)
- Best practices still required (malware exists for macOS, though less than Windows)

### Third-party Security Software
- Antivirus and anti-malware available from third parties
- May require increased update frequency for latest protection

> **Exam Reality:** macOS is not immune to malware – security best practices apply to all OSes.

---

## 7. macOS Updates

### App Store Updates (Applications)
- App Store → Updates tab
- Updates for applications installed via App Store

### Software Update (Operating System)
**Location:** System Settings → General → Software Update

| Feature | Description |
|---------|-------------|
| **Download updates** | Automatically when available |
| **Install macOS updates** | Core operating system updates |
| **App Store updates** | Automatic application updates |
| **Security responses & system files** | Critical security patches (turned on by default) |

### Beta Updates
- Updates in testing phase (not fully released)
- **Not turned on by default**
- **Caution:** Can cause system problems

---

## 8. Rapid Security Response (RSR)

| Feature | Description |
|---------|-------------|
| **Purpose** | Emergency security patches (not monthly) |
| **Default** | Enabled (install security responses & system files) |
| **Indicator** | Letter appended to OS version number indicates RSR applied |
| **Also available in** | iOS, iPadOS |

**Example:** macOS 13.4 → after RSR → macOS 13.4 (a) or similar letter designation

**Disable option:** Under Automatic Updates → Install security responses and system files

## 9. Quick Reference: macOS vs. Windows

| Function | Windows | macOS |
|----------|---------|-------|
| Executable file | `.exe` | `.app` (bundle) |
| Installer | `setup.exe`, `.msi` | `.pkg` or drag `.app` |
| Disk image | `.iso` | `.dmg` |
| Uninstall | Programs and Features | Delete `.app` from Applications |
| Program location | `C:\Program Files` | `/Applications` |
| User folder | `C:\Users\username` | `/Users/username` |
| System files | `C:\Windows` | `/System` |
| App data (user) | `AppData` (hidden) | `~/Library` (hidden) |
| Backup | File History, Backup & Restore | Time Machine |
| App store | Microsoft Store | App Store |
| OS updates | Windows Update | Software Update (System Settings) |
| Identity | Microsoft Account | Apple ID |

## 10. Sample CompTIA-Style Questions

> *A macOS user wants to uninstall an application they downloaded from the internet. What is the standard method?*
>
> A. Run the uninstaller from Control Panel  
> B. Delete the `.app` file from the Applications folder  
> C. Right-click the `.dmg` file and select Uninstall  
> D. Use the App Store to remove the application  

**Answer:** B (Delete the `.app` bundle – no formal uninstaller needed)

> *A technician needs to access application support files for a specific user on macOS. Where are these files typically stored?*
>
> A. `/Library`  
> B. `/System/Library`  
> C. `~/Library`  
> D. `/Applications/Library`  

**Answer:** C (`~/Library` – user's home directory Library folder, hidden by default)

> *An organization wants to manage macOS devices using the same Active Directory credentials as their Windows computers. What should they use?*
>
> A. Apple ID  
> B. Managed Apple ID with Apple Business Manager  
> C. Time Machine with AD integration  
> D. Rapid Security Response  

**Answer:** B (Managed Apple ID integrates with Active Directory via Apple Business Manager)

> *A macOS user receives a critical security patch outside the normal monthly update cycle. What feature delivered this patch?*
>
> A. Time Machine  
> B. Beta Update  
> C. Rapid Security Response (RSR)  
> D. App Store Update  

**Answer:** C (RSR pushes emergency security patches immediately)

> *Which macOS folder is the equivalent of `C:\Program Files` on Windows?*
>
> A. `/System`  
> B. `/Users`  
> C. `/Applications`  
> D. `/Library`  

**Answer:** C (`/Applications` contains installed application bundles)

> *What does the `.dmg` file extension indicate on macOS?*
>
> A. An application bundle  
> B. A disk image (mountable)  
> C. An installer package  
> D. A compressed archive  

**Answer:** B (`.dmg` is an Apple Disk Image – mounts like a drive)

|----------|-----------------|
| Install new software | Use `.pkg` installer or drag `.app` to Applications |

| Testing pre-release OS | Beta updates (off by default, use with caution) |
| Update applications | App Store → Updates tab |
| Update operating system | System Settings → General → Software Update |
